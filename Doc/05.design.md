# Work Management / Time Tracking Template

# 05 - Frontend Architecture & Design System

> Objetivo: Definir a identidade visual, arquitetura do frontend, componentes reutilizáveis e padrões de interface do template Work Management da MasIA.

---

# Filosofia

O objetivo não é criar uma interface futurista.

O objetivo é transmitir confiança.

O usuário deverá sentir que está utilizando uma ferramenta de trabalho profissional.

Palavras-chave:

- Premium
- Enterprise
- Minimalista
- Moderna
- Organizada
- Leve
- Confiável

Inspirado em:

- Linear
- Vercel Dashboard
- Stripe
- Raycast
- Notion Calendar
- Arc Browser

---

# Filosofia Visual

A interface deverá utilizar poucos elementos.

Muito espaço em branco (mesmo no dark mode).

Poucas cores.

Muito contraste entre conteúdo e ações.

O verde será reservado para ações positivas.

O azul será utilizado para transmitir profundidade.

---

# Identidade

Ao invés do tema "Cyber Green", o template utilizará um conceito mais próximo de um SaaS premium.

Visual:

- Dark Slate
- Soft Blue
- White
- Emerald Green

A sensação deve ser:

> "Sistema profissional utilizado todos os dias."

---

# Sidebar

Objetivo:

Ser extremamente limpa.

Características:

- largura fixa
- logo superior
- navegação principal
- navegação secundária
- perfil do usuário no rodapé

Item ativo

```css
background: rgba(255, 255, 255, 0.06);

border: 1px solid rgba(255, 255, 255, 0.08);
```

Hover

```css
background: rgba(255, 255, 255, 0.04);
```

---

# Header

Sempre possuirá:

- Breadcrumb
- Título
- Pesquisa
- Ações rápidas
- Perfil

---

# Cards

Todos os cards utilizarão o mesmo padrão.

```css
border-radius: 20px;

padding: 24px;

border: 1px solid var(--border);

background: var(--surface);
```

Hover

```css
transform: translateY(-2px);
```

Sombras extremamente discretas.

---

# Botões

Existirão apenas quatro variações.

Primary

Verde.

Secondary

Azul.

Ghost

Sem fundo.

Danger

Vermelho.

Nenhuma outra variação deverá existir.

---

# Inputs

Todos os inputs deverão compartilhar o mesmo componente.

Características:

- altura consistente
- bordas arredondadas
- foco suave
- animações discretas

---

# Tabelas

As tabelas serão um dos principais componentes do sistema.

Características:

- Cabeçalho fixo
- Linhas altas
- Hover suave
- Paginação
- Busca
- Filtros

Sem excesso de bordas.

---

# Formulários

Todo CRUD seguirá exatamente o mesmo padrão.

Estrutura:

```text
Título

Descrição

Campos

Ações
```

Nenhuma tela criará formulários diferentes.

---

# Animações

Objetivo:

Dar sensação de fluidez.

Nunca chamar atenção.

Duração:

150~250ms

Utilizar apenas:

- Fade
- Scale
- Slide curto

Evitar:

- Bounce
- Rotações
- Efeitos exagerados

---

# Componentes Base

Todo o sistema deverá reutilizar os mesmos componentes.

Exemplos:

- Page Header
- Stat Card
- Data Table
- Empty State
- Search Input
- Filters Bar
- Form Section
- Modal
- Confirm Dialog
- Badge
- Avatar
- Timeline Item
- Loading Skeleton

Nenhuma entidade deverá reinventar esses componentes.

---

# Filosofia de Desenvolvimento

Sempre que uma nova tela for criada, ela deverá reutilizar componentes existentes.

A prioridade será:

1. Reutilizar.
2. Estender.
3. Criar novo componente.

Nunca iniciar um componente do zero sem verificar se já existe algo semelhante.

---

# Experiência do Usuário

O usuário deve sentir que:

- Tudo está no lugar certo.
- Todas as telas seguem o mesmo padrão.
- Não precisa reaprender a interface.
- O sistema responde rapidamente.
- A navegação é previsível.

A interface deve desaparecer para que o foco permaneça no trabalho.

---

# Objetivo Final

O template deve parecer um produto SaaS pronto para comercialização.

Mais importante do que possuir muitas funcionalidades é transmitir consistência visual, organização e qualidade em cada detalhe da experiência.
